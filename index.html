INFO-6120 — Databases: Intro

Yoober — Ride-Sharing Relational Database

Designed a fully normalised relational database for a ride-sharing platform from scratch. Mapped out the business domain — users, drivers, passengers, rides, vehicles, licences, availability schedules, addresses, favourite locations, and a two-sided review system — then translated it into a 13-table schema with enforced foreign key relationships throughout. The design uses a subtype pattern (drivers and passengers as specialisations of accounts) to avoid data duplication. Populated with sample data and validated with queries checking referential integrity and business logic constraints across all relationships.
